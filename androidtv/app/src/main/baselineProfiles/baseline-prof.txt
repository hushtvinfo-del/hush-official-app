# HushTV — Baseline Profile (Human-Readable Format)
#
# Tells ART (Fire Stick / Android TV / Android Auto runtime) which
# methods to AOT-compile at install time. Without this, those methods
# get JIT-compiled on the fly during cold start, which is the #1
# reason "first launch after update" feels sluggish on weak ARM SoCs
# like the Fire Stick 4K.
#
# Format crash-course:
#   H  = Hot method (called many times during normal use)
#   S  = Startup method (called during app cold-start)
#   P  = Post-startup method (called shortly after the splash)
#   L  = Standard JVM type descriptor prefix (always present)
#   ** = Wildcard — match any method on the class
#
# AGP 8.5 + R8 automatically translate the original class names below
# to their R8-minified equivalents when building the release APK,
# using the same mapping file emitted at /var/www/hushtv/backups/
# mapping-canada-1.44.96.txt.
#
# When generating an UPDATED profile from a Fire Stick run
# (Macrobenchmark module — see /app/_buildenv/baseline-profile-howto.md
# once we add it), append the generated rules here. Order doesn't
# matter; duplicates are de-duplicated automatically.
#
# Library-shipped profiles (Compose, Media3, Coil, Activity, etc.)
# are merged in automatically by AGP — no need to repeat them here.
